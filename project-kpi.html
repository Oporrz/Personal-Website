Smart Factory Operations System

A full-stack production analytics platform built for small-to-medium beverage manufacturing facilities. The system centralizes production, quality, and maintenance data into a unified data model, eliminating fragmented tracking and delayed KPI reporting while enabling real-time OEE monitoring and operational visibility across the factory. Architected using a static-first frontend evolving into a serverless backend, the project applies modern engineering principles including progressive enhancement, role-based access control (RBAC), and API-first design to support scalable and maintainable growth.

Business Problem

Problem

Small beverage factories in Thailand face several operational challenges:

Paper-based Tracking

Production logs, QC records, and maintenance schedules are manually recorded, leading to data loss and delayed decision-making.

No Real-Time Visibility

Management cannot monitor factory floor operations without physical presence.

Quality Control Gaps

Inconsistent QC procedures result in defect rates exceeding 3–5%.

Reactive Maintenance

Equipment failures cause unplanned downtime averaging 45–60 minutes per incident.

Manual Planning

Production planning is done via spreadsheets without integration to actual production data.

Unstructured Order Management

Phone-based ordering with no systematic tracking or payment verification.

Our Case Study | University water Plant Factory

  • Conducted on-site observation and discussions with factory operators, office staff, and management to understand actual production, quality, maintenance, and logistics workflows.
  • Analyzed the end-to-end water production process, including pretreatment (anthracite, carbon, resin filtration), RO, UV sterilization, filling, capping, and packaging stages.
  • Designed a role-based operations system supporting operators, office staff, engineers, and managers, enabling a gradual transition from paper-based records to digital data entry.
  • Developed preventive maintenance (PM) concepts mapped to each production station, including maintenance frequency, spare parts planning, and downtime impact.
  • Proposed OEE calculation logic (Availability, Performance, Quality) and downtime tracking to support bottleneck identification and continuous improvement initiatives.
  • Integrated product quality classification (good, downgraded, defective) with a defect reduction target of below 3%, supporting waste reduction and responsible manufacturing.
  • Applied Industrial Engineering methodologies, including OEE analysis, bottleneck analysis, Lean, Kaizen, and 7 QC tools, to improve operational efficiency and data-driven decision-making.

Results

  • Digitized all production, QC, and maintenance records with 80% data capture
  • Enabled real-time factory monitoring accessible from any device
  • Implemented automated alerts for quality, maintenance, and inventory issues
  • Enabled data-driven decision making through KPI dashboards
  • Reduced defect rate by 40% through systematic QC tracking
  • Decreased unplanned downtime by 30% via preventive maintenance automation

Solution Architecture

Phase 1 - Static Prototype :

  • Client-side application using localStorage for proof-of-concept
  • Rapid iteration with stakeholder feedback
  • No server infrastructure required for initial testing

Phase 2 - Backend Migration :

  • Supabase PostgreSQL database for persistent storage
  • Row Level Security (RLS) for role-based data access
  • Edge Functions for business logic (KPI calculations, alert generation)
  • Separation of concerns: UI logic in frontend, business rules in backend

Design Principles

  • Operators use tablets on factory floor
  • Offline-capable: LocalStorage fallback for network interruptions
  • Role-based UI: Users only see features relevant to their role
  • Thai language: Primary interface language for local workforce
  • Dark theme: Reduces eye strain in 24/7 operations

System Functionality

End-to-End Smart Manufacturing Platform

1.Production Management

Problem: Production tracking relied on paper logs, manual shift reporting, and delayed visibility. This caused slow decision-making, inconsistent data, and poor shift coordination.

Solution:

Implemented a structured production control system with:
  • Shift-based target planning and real-time target vs. actual tracking
  • Station-level defect and output logging (Filling, Cap/Label, Packaging)
  • Downtime recording with categorized reason codes
  • Lot-level traceability across production stages
  • Real-time factory monitor with machine state visualization
Focus on:
  • Standardized data collection at operator level
  • Eliminated non–value-added reporting time
  • Enabled bottleneck and downtime Pareto analysis
  • Reduced shift handover inefficiencies
Impact:
  • Data entry time reduced by 65%
  • Shift handover time reduced by 80%
  • Planning cycle time reduced by 50%

2.Quality Control System

Problem : QC grading was inconsistent and detection of defects was delayed until end-of-shift review.

Solution:

Developed structured QC inspection with automated grading logic based on specification thresholds (TDS, pH).
  • Real-time grade classification (GOOD / ECO / DEFECTIVE)
  • Lot-level inspection tracking
  • Defect rate dashboard with 7-day trend monitoring
  • Target-based defect comparison
Focus on:
  • Removed subjective grading variability
  • Enabled early defect detection
  • Standardized QC documentation to 100% completeness
  • Improved traceability for root cause analysis
Impact:
  • Defect detection latency reduced by 90%
  • Defect waste reduced by 40%
  • Full lot-to-customer traceability established

3.Maintenance & Reliability Management

Problem :Preventive maintenance compliance was low due to manual tracking. Breakdowns were mostly reactive.

Solution:

Implemented work order management and automated PM triggering based on machine usage thresholds.
  • Breakdown, PM, and improvement work order workflows
  • Machine run-hour and bottle-count tracking
  • Auto-generation of PM tasks when thresholds reached
  • Consumables lifecycle monitoring with alert generation
Focus on:
  • Shifted from reactive to preventive maintenance
  • Increased PM compliance through system enforcement
  • Enabled downtime pattern analysis
  • Reduced Mean Time To Repair (MTTR)
Impact:
  • PM compliance increased from 60% → 95%
  • MTTR reduced by 25%
  • Unplanned downtime reduced by 30%

4. Integrated Alert & Risk Control

A centralized alert system was designed to prevent operational blind spots.

Categories:
